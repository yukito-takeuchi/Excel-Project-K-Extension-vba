## 既存システムの仕組み

### 全体アーキテクチャ

```
FreeWay（会計ソフト）
    ↓ CSV出力（月次）
0全科目月次データ出力.xlsm  ← データハブ（全ての中心）
    ↓ Excelの外部参照数式（自動連動）
1〜13番の各Excelファイル   ← 各種帳表・グラフを生成
```

### 0番ファイルの構造

| シート | 役割 |
|--------|------|
| Sheet(1)「602全科目月次データ出力（三期分）」 | 三期分CSVの貼り付け先 |
| Sheet(3)「必須入力」 | 月・利益率などを手入力 |
| Sheet(4)「602全科目月次データ出力（当期のみ）」 | 当期CSVの貼り付け先（必須） |
| Sheet(6)「税込データ専用」 | 税込CSVの貼り付け先（税抜方式のみ） |

### CSVの種類と方式

| 方式 | 三期分CSV | 当期CSV | 税込CSV |
|------|----------|---------|---------|
| 税抜方式 | Sheet(1)へ | Sheet(4)へ | Sheet(6)へ |
| 税込方式 | Sheet(1)へ | Sheet(4)へ | **不要**（数式で自動連動） |

### 外部参照リンクの実態

全ファイルのリンク先が絶対パスで固定されている：
```
C:\Users\黒永会計テレワークPC４\Desktop\FreeWay月次ｾｯﾄ\0全科目月次データ出力.xlsx
```
→ **フォルダを移動するとリンクが切れる**
→ **ファイル名を変えるとリンクが切れる**
→ 全ファイルを同フォルダに置く必要がある

## 既存マクロの分析結果（判明分）

### 共通パターン

**全マクロが同じ処理をしている：**
```
利益がプラスかマイナスかを判定
→ 対応するシートの行の高さ（RowHeight）を調整
→ グラフの見た目を整形
→ 最後に「ラベル位置を調整してください」とMsgBoxで手動作業を促す
```

**つまり既存マクロは「グラフの見た目を整えるだけ」の処理。データ集計や計算はしていない。**

### 4番（月次収益分析表）
- `ストラック図作成()` 1つのみ
- 単PL・累PL・BSシートの行高さ調整
- 最後に手動ラベル調整を促す

### 5番（三期比較決算図表）
- `三期連続比較表作成()` 1つのみ
- ①②③PL・BSシートの行高さ調整（3期分）
- Application.Calculation = Manual で高速化あり

### 6番（経営計画・予実グラフ）
- Module1：変数宣言のみ（処理なし）
- Module2：`経営計画図表作成()` がメイン
- ①②③PLシートの行高さ調整

## 毎月の手動作業フロー（現状）

```
1. 0番ファイルを開く
2. 使うファイル（4・5・6・11・12番等）も全部開く
3. FreeWayでCSV出力（最大3種類）
4. 手動コピペ × 3回（← 今回の自動化対象）
   ① 三期分CSV → Sheet(1)
   ② 当期CSV → Sheet(4)
   ③ 税込CSV → Sheet(6)（税抜方式のみ）
5. 0番「必須入力」シートで月・利益率を入力
6. 4番を開いてマクロ実行 → STRAC図整形
7. 5番を開いてマクロ実行 → 三期比較整形
8. その他使うファイルのマクロを順番に実行
9. ラベル位置の手動調整（各マクロ完了後に毎回発生）
```
