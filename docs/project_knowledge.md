## 既存システムの仕組み

```
FreeWay（会計ソフト）
    ↓ CSV出力（月次）
0全科目月次データ出力.xlsm  ← データハブ（全ての中心）
    ↓ Excelの外部参照数式（自動連動）
1〜13番の各Excelファイル   ← 各種帳表・グラフを生成
```

| 方式 | 三期分CSV | 当期CSV | 税込CSV |

| 税抜方式 | Sheet(1)へ | Sheet(4)へ | Sheet(6)へ |

### 外部参照リンクの実態

全ファイルのリンク先が絶対パスで固定されている：
```
C:\Users\黒永会計テレワークPC４\Desktop\FreeWay月次ｾｯﾄ\0全科目月次データ出力.xlsx
```
→ **フォルダを移動するとリンクが切れる**
→ **ファイル名を変えるとリンクが切れる**
→ 全ファイルを同フォルダに置く必要がある

## 既存マクロ 完全解析（全8ファイル）

### RowHeight パターンの仕組み（全マクロ共通）

```
Rows("4").RowHeight = Cells(4, 75)
```

この1行が全マクロの核心。意味：
- 行4の高さ（ポイント単位）を、セル(4, 75)の**数値**にセットする
- 列75 = BW列（画面右端はるか外）に、計算済みの高さ値が数式で入っている
- つまり「Excelのグラフ要素（行の色付きセル）を財務データに連動してリサイズする」仕組み
- 利益がプラス/マイナスで対象行が違うため、毎月マクロを実行する必要がある

### パターン別分類

#### グループA：RowHeight調整マクロ（4・5・6・8・12・13番）

全て同じ流れ：
1. 利益/負債の基準セルを判定（`>= 0` か否か）
2. 対応するPL/BSシートを `Activate`
3. 複数行に `RowHeight = Cells(行, 列)` を設定
4. 利益マイナスの場合は特殊処理（既存図形削除 → セル範囲をリンク貼り付け）
5. 最終シートに移動
6. `MsgBox "ラベル位置を調整してください。"`

**重要：全マクロ最後のMsgBoxにより、完全無人自動化は不可能。毎回ラベルの手動調整が必要。**

#### グループB：特殊マクロ（10・11番）

10番は手入力削除、11番はCF図解の動的生成。後述参照。

### 4番：月次収益分析表BS経営分析表（`4_月次収益分析表BS経営分析表.bas`）

**マクロ：** `ストラック図作成()`

**参照シート：** `STRAC入力`

**処理対象シート：** 単PL・累PL・BS（3シート）

**判定セル：**
- 単PL用：`STRAC入力.B23`（単月利益）
- 累PL用：`STRAC入力.C23`（累計利益）
- BS用：`STRAC入力.B34`（純資産）

**特徴：**
- マイナス時は `DrawingObjects` をループして既存図形を削除してから貼り直し
- `Sheets("STRAC図").Select` で最終シートに遷移

### 6番：経営計画・予実グラフ（`6_経営計画予実グラフ.bas`）

**モジュール構成：**
- Module1：変数宣言のみ（BS分析用の変数、実際には処理なし）
- Module2：`経営計画図表作成()` がメイン

**参照シート：** `入力ｼｰﾄ`（全角スペースなし、半角カナ混在に注意）

**処理対象シート：** ①②③PL（3シート）＋ 経営計画 or 経営計画(繰欠あり)

**判定セル：**
- ①PL用：`入力ｼｰﾄ.H21`、②PL：`AC21`、③PL：`AN21`

**特徴：**
- 他と違い `BS` シートの処理なし（PL3枚のみ）
- Module1の変数（当座資産・流動比率等）は宣言だけで使われていない（未完成の可能性）

### 11番：キャッシュフロー計算書（`11_キャッシュフロー計算書.bas`）

**最も複雑なマクロ。動的にCF図解シートを生成する。**

**マクロ：** 4つ（`create_zukai` / `delete_row` / `color` / `Sheet_Search1` / `Sheet_Search2`）

#### `create_zukai()` の処理フロー

```
1. アクティブシート名を取得（累計 or 単月）
2. MsgBox で「前回分を保存するか？」確認
3. 既存の「図解_XXX」シートがあればエラー停止（Sheet_Search1）
4. テンプレートシート（コピー用_累計 or コピー用_単月）をコピー
5. 新シートに「図解_XXX」と命名
6. 最大値を Cells(1, 61) から取得（= BE1、計算済み）
7. データなし行を削除（delete_row）：
   - 営業活動セクション（行18〜小計まで）
   - 投資活動セクション（Find("投資活動に使用した")）
   - 財務活動セクション（Find("財務活動に使用した")）
8. 各セクションの小計行の括弧表記を整形（▲ or + 表示）
9. workシートのセルA1〜A30に書かれたキーワードで各行を検索
10. color() サブルーチンで色付け
```

#### `color()` の色付けロジック

```
金額が + → 緑（12611584）、フリーCFなら黄（16777093）
金額が − → 赤（255）、フリーCFなら黄（16777093）

バー幅 = Round(60 × (金額/最大値), 0) セル分
  1〜20セル：1行使用
  21〜40セル：1行挿入して2行に分割
  41セル以上：2行挿入して3行に分割

バーは列36（AJ列）から右方向、または左方向（マイナス時）
```

**特徴：**
- 唯一「新シートを動的生成」するマクロ
- `work` シートにキーワードリストがある（A1〜A30に科目名）
- `前回分` シートへの保存機能あり
- セル結合・UnMerge・再結合を多用（RowHeightではなく列幅でビジュアル表現）

**マクロ：** `ストラック図作成()`

**参照シート：** `入力`

**処理対象シート：** 黒字PL・黒字BS・（条件付きで）優良PL・優良BS

**判定セル：**
- `入力.C3 > 0`：優良企業データあり → 優良PL/BSも整形、印刷範囲を2列分拡張

**特徴：**
- `PageSetup.PrintArea` の設定が含まれる唯一のマクロ
  - 優良企業あり：`"$A$1:$R$70,$S$1:$AJ$70"`（2ページ分）
  - 優良企業なし：`"$A$1:$R$70"`（1ページ分）
- RowHeight値の参照列が列14（N列）と列9（I列）← 他番号と異なる

## 毎月の手動作業フロー（現状）

```
1. 0番ファイルを開く
2. 使うファイル（4・5・6・11・12番等）も全部開く
3. FreeWayでCSV出力（最大3種類）
4. 手動コピペ × 3回（← 今回の自動化対象）
   ① 三期分CSV → Sheet(1)
   ② 当期CSV → Sheet(4)
   ③ 税込CSV → Sheet(6)（税抜方式のみ）
5. 0番「必須入力」シートで月・利益率を入力
6. 4番を開いてマクロ実行 → STRAC図整形
7. 5番を開いてマクロ実行 → 三期比較整形
8. その他使うファイルのマクロを順番に実行
9. ラベル位置の手動調整（各マクロ完了後に毎回発生）
```

## 改善方向性と制約

### できること（自動化可能）
- **CSV取込** → `CSV自動取込.bas` で実装済み
- **各マクロの一括実行** → 0番に `Application.Run "'4番.xlsm'!ストラック図作成"` を並べるだけで可能

### できないこと（構造的制約）
- **ラベル位置の自動調整** → 全マクロが `MsgBox "ラベル位置を調整..."` を出す設計
  - ラベル（テキストボックス・図形）はVBAで自動配置できるが、クライアント側の確認フローとして残す可能性が高い
  - 現時点では「自動化スコープ外」と判断

### 改善候補（要クライアント確認）
